# TEMPLATE BASE DE PROJETO NO GITHUB

## Objetivo deste repositório
Este repositório foi criado como modelo para ajudar as equipes a organizar, desenvolver e documentar seus projetos no GitHub.

Cada grupo deverá adaptar este modelo ao tema do seu próprio projeto.

---

## Estrutura do repositório

- `projeto/` → pasta onde ficará o desenvolvimento do site ou sistema
- `documentos/` → pasta onde ficará a documentação do projeto
- `README.md` → apresentação geral do projeto no GitHub

---

## O que deve existir na pasta `projeto/`

A pasta `projeto/` deve guardar os arquivos do sistema ou site desenvolvido pela equipe.

Exemplo para um projeto simples em HTML, CSS e JavaScript:
- `index.html`
- `style.css`
- `script.js`

Dependendo do projeto, essa pasta também pode conter:
- imagens
- páginas extras
- arquivos de componentes
- pastas organizadas por telas ou funções

---

## O que deve existir na pasta `documentos/`

A pasta `documentos/` deve guardar os registros do projeto.

Ela pode conter:
- `Projeto_Equipe.docx`
- `Projeto_Equipe.pdf`
- `planejamento.md`
- `requisitos.md`
- `diario-de-bordo.md`
- `testes.md`
- `referencias.md`
- `evidencias/`

---

## Estratégia de branches

Este projeto utilizará a seguinte organização de branches:

- `main` → versão principal do projeto
- `develop` → versão em desenvolvimento
- `feature/nome-da-tarefa` → branch criada para uma tarefa específica

### Exemplos de feature
- `feature/readme-inicial`
- `feature/estrutura-projeto`
- `feature/documentacao-base`
- `feature/tela-inicial`
- `feature/formulario-cadastro`
- `feature/testes-iniciais`

---

## Como a equipe deve trabalhar

1. Criar ou escolher uma tarefa no GitHub Projects
2. Criar uma branch `feature/...` a partir da `develop`
3. Fazer a alteração nessa branch
4. Realizar commit
5. Abrir pull request para a `develop`
6. Registrar evidências da tarefa concluída

---

## Exemplos de tarefas principais para desenvolver um website

Se o projeto for um website, a equipe pode organizar tarefas como:

- criar o README inicial
- criar a estrutura de pastas
- desenvolver a tela inicial
- criar menu de navegação
- criar formulário
- estilizar a página
- adicionar imagens
- testar funcionamento
- corrigir erros
- atualizar documentação

### Exemplos de branches para website
- `feature/tela-inicial`
- `feature/menu-navegacao`
- `feature/formulario-contato`
- `feature/estilo-pagina`
- `feature/testes-site`

---

## Exemplos de tarefas principais para desenvolver um sistema/projeto

Se o projeto for um sistema, a equipe pode organizar tarefas como:

- definir problema e objetivo
- levantar requisitos
- criar protótipo das telas
- modelar casos de uso
- desenvolver cadastro
- desenvolver listagem
- implementar busca
- realizar testes
- registrar evidências
- finalizar documentação

### Exemplos de branches para sistema
- `feature/requisitos-iniciais`
- `feature/prototipo-telas`
- `feature/cadastro-usuarios`
- `feature/listagem-dados`
- `feature/testes-finais`

---

## O que será avaliado

A equipe deverá demonstrar:
- organização do projeto no GitHub
- uso adequado das branches
- desenvolvimento do projeto na pasta `projeto/`
- documentação correta na pasta `documentos/`
- registro de tarefas no GitHub Projects
- evidências do processo de desenvolvimento

---

## Orientação final

Este template é apenas uma base.
Cada grupo deverá adaptar o nome do projeto, os arquivos, as tarefas e as funcionalidades de acordo com a proposta escolhida.
